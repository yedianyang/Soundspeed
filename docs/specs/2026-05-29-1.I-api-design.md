## 关键决策

### 1. take 端点 `async def` + 直接同步 publish
- **why**：`take.end` 的 fire-and-forget L2 靠 `asyncio.get_running_loop()`（`backend/core/orchestrator.py:214`，且仅在注入 `llm_service`/`l2_runner` 时走到）。若端点是 `def`，FastAPI 丢线程池跑 → 无 running loop → `get_running_loop()` 抛错被 `publish()` 吞掉（orchestrator.py:79-82）→ 生产 L2 静默不触发。裸 Orchestrator（无注入 deps）的测试在依赖守卫处 return，照不到这条路径。
- 不能把 `publish` offload 到线程池（会丢 loop ref）。接受 SQLite 同步写带来的短暂 loop 阻塞（此规模可接受）。
- **防回归**：测试注入 stub `l2_runner` 断言 `take/end` 后 `_l2_task` 被创建。代码注释写明 why，防被「简化」回 `def`。

### 2. WS 同步↔异步桥接
- `ConnectionManager.broadcast(topic, payload)` 做成 **topic 无关**（`asr.*` 与 `take.changed` 共用一个 seam）。
- 机制：存 event loop 引用 + `run_coroutine_threadsafe`（同步 handler 可能从 ASR 线程触发）。
- frozen dataclass payload 用 `asdict()` 序列化。
- **边界处理**（codex 要求）：loop 未设置 / 已关闭；`run_coroutine_threadsafe` 返回的 Future 异常不丢弃；broadcast 中途连接 disconnect；广播时连接集合被并发修改（用快照迭代）。
- **事件来源**：`take.changed` 来自 Orchestrator 的 take handler（已实现，0 依赖 1.A），现在就能端到端测；`asr.*` 来源是 1.A/1.C，1.I 只负责「订阅 + 转发」，用合成 `AsrFinalPayload` 测桥接。

### 3. 请求契约
- ts 由服务端 `time.time()` 生成。
- `POST /api/v1/take/start` body = `{scene_id:int, shot:str|null}`；`POST /api/v1/take/end` body 空。
- 映射到 `TakeStartPayload(scene_id, shot, start_ts)` / `TakeEndPayload(end_ts)`（`backend/core/events.py`）。

## 鉴权（architecture §10.3）
- ADMIN_TOKEN：启动从环境读，缺失则生成随机并打印 console（hackathon 现场友好）。
- REST：`require_admin` 依赖校验 `Authorization: Bearer <token>`（take 端点为 ADMIN）。
- WS：`/ws` query `?token=<token>` 校验。
- `/healthz` 无鉴权。`.env.example` 记 `ADMIN_TOKEN`。

## 测试（`backend/tests/test_api.py`）
- `/healthz` 返回 200。
- take 端点无 token / 错 token → 401；正确 token → 200 且 orchestrator 收到对应事件（spy handler）。
- **take/end 注入 stub `llm_service`+`l2_runner` → 断言 `_l2_task` 被创建**（决策 1 核心，防回归）。
- `/ws` 无 token 握手被拒；带正确 token 握手成功。
- `broadcast` 从 loop 内 + 从后台线程分别投递 → 已连 WS 收到。
- `take.changed` 端到端：`POST /take/start` → 已连 WS 收到 take.changed（真实链路）。
- `asr` 转发：合成 `publish(ASR_FINAL_CH1, …)` → 已连 WS 收到（桥接验证）。
- disconnect 后连接被清理。
