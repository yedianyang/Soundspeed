## 1. 起后端

worktree 根目录，用项目 `.venv`（Python 3.12），**设固定 `ADMIN_TOKEN`**（否则每次重启随机生成，前端存的 token 会失效，要重填）：

```bash
cd /Users/yedianyang/Documents/GitHub/Soundspeed/.claude/worktrees/feat+1.J-1.L-frontend-admin
SOUNDSPEED_DEV=1 ADMIN_TOKEN=devtoken SOUNDSPEED_DB=./soundspeed_dev.db \
  GEMMA_MODEL_PATH=/Users/yedianyang/Documents/GitHub/Soundspeed/models/gemma-4-E4B-it-Q4_K_M.gguf \
  .venv/bin/python -m backend.api
```

- 默认 `0.0.0.0:8000`。控制台打印 token。
- `SOUNDSPEED_DEV=1`：自动播种一个 active scene（Scene_1）+ 挂 `/api/v1/debug/asr` 注入端点。
- **`GEMMA_MODEL_PATH` 必给绝对路径**：worktree 的 `models/` 是空的（权重 gitignored，不随 worktree 带过来），不设这个 L2 会报 `Model path does not exist` 并降级（take 照常进列表但无摘要）。指向主仓库 `models/gemma-4-E4B-it-Q4_K_M.gguf`（那是个指向 HF 缓存的 symlink）。
- 健康检查：`curl http://localhost:8000/healthz` → `{"status":"ok"}`。

## 3. 连后端（填 token）

1. 打开 `http://localhost:5173/admin`，点右上齿轮（设置）。
2. 「服务器连接」段：API base 默认 `http://localhost:8000`（如后端不在本机改这里，对应改前端 `VITE_API_BASE`）；admin token 填 `devtoken`，保存。
3. 保存后连接状态应变为 open；保存会触发 scenes/takes 重取，REC 按钮在拿到 active scene 后启用。

## 4. 录一条 take（验 1.K + 1.J + 1.L）

1. **按 REC**（红圆）。状态栏显示「当前场次 Scene_1 / Take 1 / 录制中」。（1.K：take_number 由 take.start 触发的 take.changed 绑定。）
2. **注入转录**（1.C 前替代真实 ASR）。另开终端：
   ```bash
   T=devtoken; U=http://localhost:8000/api/v1/debug/asr
   # ch1 final，speaker_0
   curl -s -XPOST $U -H "Authorization: Bearer $T" -H "Content-Type: application/json" \
     -d '{"ch":1,"text":"你昨天为什么没告诉我真相","speaker":"speaker_0","is_partial":false}'
   # ch1 partial（灰斜体）
   curl -s -XPOST $U -H "Authorization: Bearer $T" -H "Content-Type: application/json" \
     -d '{"ch":1,"text":"因为我","speaker":"speaker_1","is_partial":true}'
   # ch2 final（speaker 置空，灰）
   curl -s -XPOST $U -H "Authorization: Bearer $T" -H "Content-Type: application/json" \
     -d '{"ch":2,"text":"boom 杂音","speaker":null,"is_partial":false}'
   ```
   Live 面板应实时滚屏：final 黑、partial 灰斜体带光标，ch1 speaker 按 speaker_id 分色。（1.J）
3. **再按 REC 停止**。take.end → L2 触发。LLM chip：首条 take 因加载 Gemma 显黄点「Loading…」，加载后「L2」，完成「Idle」绿。（1.K + 懒加载需求）
4. L2 完成后（首次含模型加载，约 10–30s），History tab 出现该 take；展开看 segments + L2 摘要（script_diff_summary）+ line_matches。LLM 反馈 tab 显示摘要。（1.L）

## 5. 验收清单

- [ ] **【首要观察】LLM chip 首条 take 结束时出现黄点「Loading…」**（Gemma 懒加载），加载后「L2」，完成「Idle」绿（1.K + 用户指定需求）。这条最关键也最不确定：Gemma 首次加载在 worker 协程内同步跑会冻结 event loop，「loading」推送与 worker 阻塞谁先跑是调度竞态，headless 推不出来，必须现场看。**若黄点不出现**，按 §6 contingency 把 `_ensure_client` 挪进 `to_thread`（1.F lane）。
- [ ] Live 转录双声道滚屏，partial 灰 / final 黑，ch1 speaker 分色（1.J）
- [ ] 按 REC 状态栏出 take 号 + 录制态；停止回 idle（1.K）
- [ ] take 结束后 History 列表新增该 take；≤10s L2 摘要填入（需 Gemma 权重 + 剧本才有实质 diff）（1.L）
- [ ] LLM 反馈 tab 显示**当前这条** take 的摘要（不是上一条）——验「最新 take」选取正确（1.L）
- [ ] 无 console error；后端重启后 WS 自动重连 + take 列表重取对齐

## 6. 注意 / 已知限制

- **真实音频→ASR（1.C）未做**：转录靠 `/debug/asr` 模拟。接 1.C 后该端点可弃用。
- **默认 ASR 模型 = `medium-q8_0`**（量化版，约 fp16 一半的磁盘/RAM，换 ~1% WER；见 issue #22）。要 fp16 精度基线：起后端时加 `SOUNDSPEED_ASR_MODEL=medium`。首次切到 q8_0 时，启动后台预热会自动从 ggerganov/whisper.cpp HF 仓拉 `ggml-medium-q8_0.bin`（~770MB，需联网；现场离线先在有网处预热一次让其落 `models/whisper/` 缓存）；老的 `ggml-medium.bin` 缓存不会复用。
- **L2 首次加载 Gemma**：慢且需 `models/` 权重就位。权重缺失或无剧本时 L2 降级（script_diff=null），take 仍出现在列表，摘要显示降级文案——属预期。
- **首次加载阻塞（contingency）**：Gemma 首次加载在 worker 协程内同步跑（`backend/llm/service.py` `_worker` 里 `_ensure_client()` 不在 `to_thread`），会短暂冻结 event loop（WS/REST 顿挫），且可能让「loading」推送排在 worker 阻塞之后、黄点延迟或不显。已知风险（spec §2.2），彻底修属 1.F：把 `_ensure_client()` 调用挪进 `asyncio.to_thread`，加载不再阻塞 loop，「loading」立即送达。若验收时黄点不显，做这个改动再验。
- **scene/shot/take 切换、mark、Delete、Next take、memo、导入导出、observer、电平**：本阶段保持本地 mock，无对应端点（scope guard）。
- **token 持久化**：用固定 `ADMIN_TOKEN` 避免重启失效。

## 7. 说话人归属纠正（segment speaker correction）手动验收

前置：后端起好（同 §1）、前端起好、有至少一条含 ch1 转录的 take（按 §4 录一条）。

- [ ] History 展开一条 take，ch1 segment 的 speaker 现在可点（hover 有底色），点开出「切换说话人」菜单。
- [ ] 菜单候选 = 本 take 出现过的 speaker（含当前值，当前值高亮）+「未知」，**不是** SZA/YY/Unknown。
- [ ] 选另一个 speaker → 列表即时更新为新值；刷新页面后仍是新值（已落库）。
- [ ] 选「未知」→ 该条 speaker 变为不显示（PATCH body 是 `null`，非字符串 "未知"）。
- [ ] 纠正后，L2 区上方出现小字「说话人已纠正，剧本分析未更新」；折叠再展开同一 take，提示仍在（本会话持续）；刷新后消失。
- [ ] ch2 segment（若有）的 speaker 不可点。
- [ ] 不同 SPEAKER_XX 颜色稳定区分（哈希分色）。
